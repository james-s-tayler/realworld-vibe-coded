## i18n Conventions

### Backend
- Standard FluentValidation messages (NotEmpty, MaximumLength, etc.) auto-translate via FluentValidation's built-in LanguageManager ‚Äî do NOT add explicit .WithMessage() for these
- Custom business-rule messages use `IStringLocalizer<SharedResource>` with resource files in `App/Server/src/Server.SharedKernel/Resources/`
- Always use `SharedResource.Keys.*` constants (generated by `Darp.Utils.ResxSourceGenerator`) ‚Äî NEVER use magic string keys like `localizer["MyKey"]`
- Validators are singletons in FastEndpoints ‚Äî MUST use lambda `WithMessage(x => localizer[SharedResource.Keys.MyKey])` for localized messages (I18N002 enforces this)
- Handler ErrorMessage assignments should use `_localizer[SharedResource.Keys.MyKey]` not string literals (I18N001 warns)
- Culture set from `Accept-Language` header via `RequestLocalizationMiddleware`
- Supported languages configured in `appsettings.json` `I18n` section
- New custom messages MUST be added to ALL .resx files ‚Äî `DarpResX005` (build error) enforces parity between default and `*.ja.resx`

### Frontend
- Translation files: `App/Client/src/i18n/locales/{lang}.json`
- Use `useTranslation()` hook ‚Äî NEVER hardcode user-facing strings
- `eslint-plugin-i18next` `no-literal-string` rule (jsx-text-only mode) errors on untranslated JSX text ‚Äî caught by `LintClientVerify`
- Carbon `translateWithId` components: pass `(id) => t('carbon.${id}')`
- Carbon Pagination: pass translated text via direct props
- Language synced from user object in `AuthContext`
- `Accept-Language` header sent on all API requests via `clientFactory.ts`
- New UI text MUST be added to ALL locale JSON files
- Translation keys use dot notation: `page.section.key`

### Adding a New Language
1. Backend: Add new `.resx` file, add language code to `I18n:SupportedLanguages` in appsettings.json
2. Frontend: Add new locale JSON file, add entry to `SUPPORTED_LANGUAGES` in constants.ts, register in `i18n.ts`
3. FluentValidation built-in messages: automatic (56+ languages supported)

### Enforcement Summary
| Rule | Scope | Severity | What it catches |
|------|-------|----------|-----------------|
| `DarpResX005` | Backend build | Error | Missing key in any .resx translation file |
| `I18N001` | Command/Query handlers | Warning | Hardcoded string in `ErrorMessage` assignment |
| `I18N002` | FastEndpoints Validators | Error | Eager `localizer["Key"]` in `.WithMessage()` (must use lambda) |
| `i18next/no-literal-string` | Frontend `.tsx` | Error | Untranslated literal text in JSX |
